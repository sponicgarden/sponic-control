Email Design Guide

Layout & Structure

All emails use a table-based layout for maximum client compatibility. Outlook desktop uses the Microsoft Word rendering engine, which ignores div-based layouts entirely.

Typography

Font sizes, line heights, and safe stacks optimized for both desktop and mobile rendering.

Spacing & Padding

Apply padding to <td> cells only — Outlook strips padding from most other elements. Always set cellpadding="0" cellspacing="0" border="0" on every table.

Buttons & CTAs

Minimum 44×44px touch target per WCAG. Use table-based buttons for Outlook compatibility; border-radius degrades gracefully to square corners.

Images

Upload at 2× resolution for retina displays. Always include meaningful alt text — many corporate Outlook clients block images by default.

Colors & Contrast

WCAG 2.2 AA compliance is required. All text must pass minimum contrast ratios against its background.

Mobile Rendering

70%+ of emails are opened on mobile. Our fluid-hybrid layout works without media queries; @media provides progressive enhancement where supported.

Dark Mode

Email clients handle dark mode inconsistently. Design defensively so emails look acceptable even when fully inverted.

Client Quirks & Limits

Key rendering differences and hard limits across major email clients.

UI Components

Settings Cards

Use for navigable settings lists, feature menus, and admin panels. Icon tile on left, title + description in center, chevron on right. Minimum 44px touch target.

Icon Tiles

Colored squares with white SVG icons. Use consistently — one color per functional area. Pair with a text label below when used standalone.

Section Headers

Bold title with optional emoji or icon prefix, muted description paragraph below. Use to introduce grouped content areas on admin pages.

Navigation List

Left sidebar or drawer navigation. Active item shows a left-side accent border. Muted icon on the left, label, optional badge on right.

App / Integration Tiles

Horizontal row of circular or rounded app logos for integration listings. Consistent size (40px), subtle border, 8px gap.
