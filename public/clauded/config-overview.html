Config Files
CLAUDE.md is always loaded (~30 lines). On-demand docs are loaded only when the task matches their trigger. This saves thousands of tokens per conversation.

For example, a database task might add SCHEMA.md and CREDENTIALS.md, while a styling task would load PATTERNS.md instead. The full doc set is rarely loaded at once.

Actual sizes depend on your project. A small project might use 50 KB total across all docs; a large one with many integrations could reach 200 KB+. The on-demand system ensures only relevant context is loaded per conversation.

Auto-loaded Always in Context

These files are automatically loaded into every Claude Code conversation. They define the core rules and behaviors.

CLAUDE.md

Always loaded
Project directives — the root of the configuration tree. Contains mandatory behaviors, code guards, and pointers to on-demand docs. Defines naming conventions, UI patterns, push workflow, and all rules Claude Code must follow.
Points to: docs/CREDENTIALS.md, docs/SCHEMA.md, docs/PATTERNS.md, docs/KEY-FILES.md, docs/DEPLOY.md, docs/INTEGRATIONS.md, docs/CHANGELOG.md

CLAUDE.local.md

Always loaded
Operator directives — private, not checked into git. Contains DB access shortcuts, push workflow overrides, live URLs, and any operator-specific instructions.
Points to: docs/CREDENTIALS.md (for DB access), version.json

version.json

Always loaded
CI-generated release metadata — updated on every push to main. Contains the current version string, Git SHA, and commit messages. Read after every push to report the deployed version.

On-demand Documentation Files

Loaded into context only when the task matches specific triggers defined in CLAUDE.md. Keeps context lean for simple tasks.

docs/CREDENTIALS.md

On-demand
Secrets & access keys — gitignored. Contains Supabase project keys, API tokens, SSH credentials, and any other secrets Claude Code needs. Loaded when running SQL, deploying functions, SSH-ing, or making API calls.

docs/SCHEMA.md

On-demand
Database schema — table definitions for all enabled features. Covers core tables, relationships, and column types. Loaded when writing queries, modifying tables, or debugging data.

docs/PATTERNS.md

On-demand
UI conventions — Tailwind CSS design tokens, Supabase query patterns, and reusable component snippets. Loaded when writing UI code, styling with Tailwind, or during code review.

docs/KEY-FILES.md

On-demand 4+ features
File directory — organized listing of project files by module. Only generated for projects with 4 or more features. Helps Claude Code navigate large codebases without scanning the filesystem.

docs/DEPLOY.md

On-demand
Deployment workflow — push steps, version format, live URLs, and post-deploy verification. Loaded when pushing, deploying, or answering version questions.

docs/INTEGRATIONS.md

On-demand If services configured
External APIs — only generated if external services are configured. Contains vendor details, cost tiers, API usage logging requirements, and integration setup instructions.

docs/CHANGELOG.md

On-demand 6+ features
Recent changes — only generated for projects with 6 or more features. Chronological log of significant changes, useful for understanding migration context or recent architectural decisions.

Settings Claude Code Configuration

JSON configuration files in .claude/ that control Claude Code's runtime behavior.

.claude/settings.json

Always loaded
Environment settings — enables TypeScript Language Server Protocol support (ENABLE_LSP_TOOL=1) for better code intelligence in Claude Code.

.claude/settings.local.json

Always loaded
Local permissions — private, not checked into git. Contains user-specific Claude Code preferences, permission rules, and MCP server configurations.

.claude/launch.json

Always loaded
Dev server configurations — defines local development servers for Claude Code's preview tools. Each entry specifies the command, port, and working directory for a dev server.

Memory Persistent Memory Files

Claude Code's persistent memory system — stored in ~/.claude/projects/, indexed by MEMORY.md. These survive across conversations.

MEMORY.md

Always loaded (first 200 lines)
Memory index — always loaded into context. Contains the master index of all memory files, data lookup routing tables, quick query templates, and important gotchas discovered across sessions.
Points to: memory/*.md (session summaries, user preferences, etc.)

memory/*.md

On-demand
Individual memory files — session summaries, service access details, infrastructure notes, user preferences, and other persistent knowledge. Loaded when referenced by MEMORY.md or when context is needed from a prior session.
